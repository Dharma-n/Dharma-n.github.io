Middleman 教程

- 什么是Middleman

Middleman 是一个用来生成静态页面的生成器，通过在本地渲染/source里面的内容，生成/build.

  • Source
    • 源代码，以及部分逻辑
  • Build
    • 静态html
    • (/Build里面的内容永远不用修改)

- Github Page

Github支持hosting静态页面，并且显示在< xxxx >.github.io这个二级域名下。

所以, 用Middleman来生成github pages是一个非常好的选择。

- 为什么是Middleman 而不是 Github 官方推荐的 Jekyll ?

  • Jekyll只支持单语言
  • Jekyll的Blog Engine不可扩展
  • Jekyll不支持HAML

- 设置Github Blogging的环境

  1. 安装bundler sudo gem install bundler
  2. 浏览到想要放置blog文件的文件夹
  3. 从repo下载源代码 git clone https://github.com/nekokami/teampage-source.git
  4. 下载完了之后会出现一个叫做”teampage-source”的文件夹, 如果想重命名：mv teampage-source your-desired-name
  5. 进入源码文件夹 cd your-desired-name
  6. 安装Gemfiles里面包含的dependency sudo bundle install
  7. 创建build文件夹 mkdir build
  8. cd build
  9. git init
  10. git remote add https://github.com/nekokami/nekokami.github.io.git
  11. git pull origin master
  12. cd ..

环境设置到此为止

- 开始Blogging

  1. 在middleman源码的目录下 middleman article "Your blog title" --lang cn 此处可以是cn, en或者是ja
  2. 在source/cn下找到your-blog-title.html.markdown文件
  3. ---
title: Middleman 教程
date: 2014-01-12 00:41 UTC
lang: cn <-在这里加入一行表示语言
tags:
---

####我是Title, 4号字

从这里开始Blogging, 这里是blog的正文
遵循Markdown标签的书写规则, 具体请参照
[点我](http://wowubuntu.com/markdown/)
  4. 保存文件, 并且在middleman的源码目录下middleman build
  5. build通过了自后cd build
  6. 提交修改 git add .
  7. Commit git commit -m "your message"
  8. 向Server Push git push origin master
  9. 大功告成

- 后记

对于没有学过编程的人来说，这些步骤可能是比较复杂。但是就常规的Blogging Routing来说，其实一共只有8个Command. 下次，将介绍如何批量执行linux command. 另外Markdown 是一个很常用的标签语言。
